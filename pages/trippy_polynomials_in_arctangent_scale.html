Trippy polynomials in arctangent scale

This is a cubic parabola.

x3 + x2 - 4x + 2

You can see that it intersects the x-axis at least two times, and it has at least one local minimum. However, this is not the whole truth. It intersects the x-axis exactly three times, and it has exactly one minimum and one maximum. The plot fails to show that. You can drag-scroll it to the left or zoom out with shift + mouse wheel to find the missing intersection. The plot is interactive, but it can only show a small piece of ℝ2 space at a time.

Let's try something different. Instead of a linear scale, let's use the arctangent scale, which displays everything from -∞ to +∞ on a limited square. Now the same plot looks like this.

Try drag, double click, and shift+mouse wheel if you're browsing from a PC.

Yes, it looks a little unconventional, but you can now see everything.

You can clearly see that the function's domain (all the possible x values) and range (all the possible y values) are both the whole ℝ. You can see that it has three intersections with the x-axis, one local maximum, and one local minimum. It starts from -∞ and goes to +∞. Arctangent scale leaves no mystery behind, it shows all the juicy stuff about this function right away.

So let's look at the polynomials using arctangent scale plots.

X-axis intersections and roots

When a function plot intersects the x-axis, it is the same as.

ax3 + bx2 + cx + d = 0

In other words, each intersection is a root of a polynomial equation with the right side being 0.

Now let's make our cubic parabola bounce up and down by oscillating its last coefficient.

x3 + x2 - 4x + d
d ∈ (0, 4)

The same plot in the arctangent scale looks like this.

What can you tell about cubic equations by observing this plot?

It looks like they have either one or three roots, but when a local minimum or a maximum touches the axis, the two roots “merge” into one. This makes the two roots possible.

Generally speaking, a polynomial equation of n-degree may have up to n roots in real numbers. With odd n, there is always one root; with even n, there may be none. We'll see why a bit later.

Now let's oscillate all the coefficiens.

ax3 + bx2 + cx + d
a ∈ (-1, 1)
b ∈ (-1, 1)
c ∈ (-1, 1)
d ∈ (-1, 1)

What new do you see?

Yes, it covers the whole range and its global behavior is governed by the top coefficient. This goes not only for the cubic parabolas but for all the polynomials. On a global scale, the highest degree member sets the tone for the whole function.

It's easy to see why it's happening. Consider this polynomial.

xk + ak-1xk-1

For every possible ak-1, there is x such as x > ak-1. So whether the function goes down or up may be defined by the ak-1 in some area (locally), but as the function travels away from this area, sooner or later x from the first member “beats” the ak-1 and drags the whole function along with xk.

Symmetric polynomials

This function is symmetric.

P(x) = x2

Of course, squaring a number “eats” up its sign so x2 = (-x)2. Also, ax2 = a(-x)2. And also (axi)2 = (a(-x)i)2. So every member with an even coefficient is symmetric. The polynomials made up of only the even degree members are symmetric.

What can you tell about symmetric polynomials by observing this plot?

Yes, if a symmetric polynomial starts from -∞ it finishes in -∞, and if in +∞ — in +∞. It may cross the x-axis an even number of times, or it may barely touch it without crossing. It may not even come close to the axis.

As the top coefficient goes through the 0, you can see that the starting and ending points switch from -∞ to +∞ and back. But what happens when the coefficient is exactly 0? A function degenerates into a constant.

There is a catch. Technically, P(x) = a is not a polynomial. But it is a function polynomials degenerate to when all the higher-order coefficients turn to 0. Then, of course, the degenerated polynomial never touches the infinity.

This is an important corner case for polynomial approximation.

Anti-symmetric polynomials

This function is anti-symmetric.

P(x) = x1

In this context, anti-symmetry means that P(x) = -P(-x). This is, of course, true for ax1 and ax2i+1.

Polynomials made of only the anti-symmetric members are anti-symmetric.

Which is true about anti-symmetric polynomials?

Polynomials are continuous. As an antisymmetric polynomial moves along the axis, it enters the y-axis in some point x = 0, y = a. But then, since it's anti-symmetric, it has to jump to -a and this is impossible unless a is 0.

The main takeaway here is that if a polynomial Pn(x) has an even degree n, it behaves globally as a symmetric polynomial. Meaning, that it starts and finishes in the same infinity and it may or may not cross the x-axis up to n times.

But if a polynomial Pn(x) has an odd degree n, it behaves globally as an anti-symmetric polynomial. It starts and finishes in different infinities and it has to cross an x-axis at least one time.

Derivatives

You only need to know three things about polynomial derivatives.

d xn = nxn-1
dx
d (f(x) + g(x)) = d f(x) + d g(x)
dx dx dx
d k f(x) = k d f(x)
dx dx

Now let's get back to our cubic parabola and put its derivative on the same plot.

What is true about the derivative of a polynomial?

A negative derivative means that the function is going down, and a positive — up. So when a derivative crosses the x-axis, the source function makes a bump. It's either a local minimum or a maximum.

Since we established that a polynomial equation of degree n has up to n roots, a polynomial of n+1 degree can have up to n bumps.

Now let's dive deeper into technicalities. A derivative of a polynomial is either a polynomial or a constant. We've seen this before, technically a constant is not a polynomial, but polynomials may degenerate or differentiate into one. A derivative of a constant is null.

Also, if a derivative only touches the x-axis but doesn't go across it, there is no local minimum or maximum on a target function. The function stops decreasing or increasing at one particular point, and then it moves its way again. This is also an important corner case for polynomial modeling since a root of a derivative usually but not necessarily indicates a bump on a source function so that's how we usually find them.

Now let's differentiate it all the way. From a cubic parabola a constant.

I know, the plot is captivating but this time please concentrate on the coefficients. What do you see?

If we differenciate ax3 + ...whatever once, we'll get 3ax2 + ...something. The second differentiation gives 6ax + ...some constant. The third one — 6a. There is no x now, only the coefficient.

Now imagine we do the same exercise but with a higher degree polynomial something... + ax3 + ...whatever. In the same three differentiations, it transforms into members with xs... + 6a. It's a polynomial, but we can turn it into a single number if we compute it in 0. Every polynomial in 0 is just its last coefficient, and the same goes to all its derivatives.

This means that every polynomial coefficient an before xn is connected to its n-th derivative in 0.

an = dn P(x) ÷ n!
dxn

And this means that a polynomial is completely defined by its derivatives in 0.

Maclaurin series and Taylor series

We can mimic some other function by measuring some of its derivatives in 0 and building a polynomial from that.

This is the exponential function in the arctangent scale.

I choose it as an example since it's the easiest thing to differentiate.

d ex = ex
dxn

In 0, the exponential function equals 1, and since it differentiates to itself, all its derivatives in 0 are 1s. How cool is that!

Ok, so the coefficient before x is the same as the first derivative in 0. Which is 1. That gives us this.

P(x) = x + 1

Well, it doesn't mimic the function well but we're just starting.

The coefficient before x2 is 1 ÷ 2.

P(x) = 1 x2 + x + 1
2

Getting closer. The coefficient before x3 is 1 ÷ (2×3).

P(x) = 1 x3 + 1 x2 + x + 1
6 2

The next one is before x4 and it's 1 ÷ (2×3×4).

P(x) = 1 x4 + 1 x3 + 1 x2 + x + 1
24 6 2

Let's do one more just to see that it still works.

P(x) = 1 x5 + 1 x4 + 1 x3 + 1 x2 + x + 1
120 24 6 2

We can go on forever but we wouldn't. Because, while the approximation or mimicking gets better with every step, it has its limitation. And knowing what we know about polynomials, we should see what it is. A polynomial always starts from ±∞. An exponent starts at 0.

We can approximate it locally with good precision. But due to completely different global behavior, a polynomial will never mimic an exponent completely.

This way of mimicking functions with polynomials is called the Maclaurin series. We can approximate other functions like this, too. A sine, a cosine, a logarithm. Well, no. We can't do the last one just yet. It isn't defined in 0.

And that's why we need the Taylor series. It's the same idea conceptually but with the Taylor series, we can build our target polynomial from the source function derivatives at any point on ℝ. Think of it as a clever hack. We move the source function so the specified point is now 0, do the Maclaurin series there, and then shift the target polynomial back.

This is especially useful given that the series approximation only works well locally. Again, we can't change the global properties of polynomials. But with the Taylor series, we can choose the locality in which we want to mimic a source function the best.
